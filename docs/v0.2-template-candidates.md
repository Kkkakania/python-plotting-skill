# v0.2 Template Candidates

Issue
[`python-plotting-skill#2`](https://github.com/Kkkakania/python-plotting-skill/issues/2)
tracks template requests after the small v0.1 catalog. This note keeps the
candidate list public and clean-room: it records chart tasks, not copied local
code or gallery assets.

## Implemented From The Shortlist

| Template | Communication task | Public-safe implementation note |
|---|---|---|
| `spectral_density` | Show frequency content for a sampled signal. | Added as a deterministic NumPy/Matplotlib template with synthetic two-tone signal data. |
| `residual_convergence` | Show iterative solver or simulation residual decay. | Added as a deterministic semilog residual plot with synthetic solver traces. |
| `main_inset` | Show a main trend with a magnified local region. | Added with Matplotlib native inset axes and deterministic synthetic signal data. |
| `shared_colorbar_panels` | Compare several heatmaps on the same color scale. | Added as four deterministic matrix panels with one shared scale and colorbar. |
| `gantt_timeline` | Show simple project tasks on a calendar-like axis. | Added as a deterministic task bar timeline using stdlib dates and Matplotlib. |

## Shortlist

The bounded v0.2 shortlist above is now implemented. Keep this issue open for
fresh, small template requests rather than widening the original list.

## Deferred

- Full Origin-style parity: too broad for this small skill.
- 3D surfaces and CFD field galleries: better kept in the larger
  `scientific-plotting-function-library`.
- Interactive Plotly output: v0.1 is deliberately static Matplotlib.
- Domain packs copied from local folders: keep those as requirements only.

## Selection Rule

Add one candidate only when it can ship as a deterministic Matplotlib template,
synthetic data, PNG/SVG gallery output, chart-selection wording, and one
repository check. If it needs private data, binary project files, or copied
figures to make sense, leave it out.
